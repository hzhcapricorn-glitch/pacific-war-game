# 战斗报告增强功能实现文档

## 功能概述

为战斗报告系统添加了详细的损伤描述，支持：
1. **重甲单位**：显示被重创次数及击沉状态
2. **空军返航**：根据制空权显示返航或坠毁
3. **损伤详情**：所有受损单位的具体情况

## 实现细节

### 1. 损失计算系统增强 (CombatSystem.js)

#### calculateLosses 函数

**旧返回值：**
```javascript
return lostCardIds; // Array
```

**新返回值：**
```javascript
return {
  lostCardIds: Array,      // 真正损失的卡牌ID
  damageDetails: Array     // 所有受损卡牌的详细信息
};
```

**damageDetails 结构：**
```javascript
{
  card: Object,           // 卡牌对象
  hits: number,           // 被选中次数
  armorValue: number,     // 重甲值
  requiredHits: number,   // 需要被选中次数（armorValue + 1）
  isDestroyed: boolean    // 是否真正损失
}
```

#### resolveCombat 函数

新增返回字段：
```javascript
return {
  // ... 原有字段 ...
  damageDetails: lossResult.damageDetails  // 新增
};
```

### 2. 能力系统增强 (AbilitySystem.js)

新增函数：
```javascript
/**
 * 检查卡牌是否有返航能力
 */
export function hasReturnToBaseAbility(card) {
  return hasAbility(card, 'return_to_base');
}
```

### 3. 战斗报告生成 (getCombatSummary)

#### 报告结构

```
🎉 战斗胜利！ / ❌ 战斗失败...

己方火力:
  对地: X / Y
  对海: X / Y
  防空: X / Y [⚠️ 不足！损失加倍]
  制空: X / Y [✓ 空军可返航]

获得奖励:
  - 补给 +X
  - 最大补给保留 +X

战损情况:
  - 单位A 被重创X次[并击沉/但未被击沉] 🛡️[💥]
  - 单位B 被击中[但成功返航/并坠毁] ✈️[💥]
  - 单位C 损失
```

#### 显示逻辑

**1. 重甲单位（armorValue > 0）**
```javascript
if (armorValue > 0) {
  if (isDestroyed) {
    // 被选中次数 >= 重甲值+1
    summary += `${card.name} 被重创${hits}次并击沉 🛡️💥\n`;
  } else {
    // 被选中次数 < 重甲值+1
    summary += `${card.name} 被重创${hits}次但未被击沉 🛡️\n`;
  }
}
```

**2. 空军单位（hasReturnToBaseAbility）**
```javascript
else if (hasReturnAbility) {
  if (airSuperiorityAchieved) {
    // 制空火力充足
    summary += `${card.name} 被击中但成功返航 ✈️\n`;
  } else {
    // 制空火力不足
    summary += `${card.name} 被击中并坠毁 ✈️💥\n`;
  }
}
```

**3. 普通单位**
```javascript
else {
  summary += `${card.name} 损失\n`;
}
```

## 示例场景

### 场景1：战列舰重创但未击沉

**配置：**
- 参战：北卡罗来纳级（重甲1）+ 2个驱逐舰
- 损失：2张

**可能结果：**
```
战损情况:
  - 北卡罗来纳级战列舰 被重创1次但未被击沉 🛡️
  - 法拉格特级驱逐舰 损失
```

**解释：**
- 战列舰被选中1次 < 2次（重甲1+1），未击沉
- 战列舰返回弃牌堆（而非商店）
- 驱逐舰被选中1次 = 1次，损失

### 场景2：战列舰被击沉

**配置：**
- 参战：衣阿华级（重甲1）+ 2个巡洋舰
- 损失：3张，防空不足加倍为6张

**可能结果：**
```
己方火力:
  ...
  防空: 3 / 8 ⚠️ 不足！损失加倍
  ...

战损情况:
  - 衣阿华级战列舰 被重创2次并击沉 🛡️💥
  - 巴尔的摩级巡洋舰 损失
  - 亚特兰大级巡洋舰 损失
```

**解释：**
- 损失加倍：3 → 6次随机选择
- 战列舰在6次中被选中2次，达到击沉条件
- 战列舰返回商店

### 场景3：空军成功返航

**配置：**
- 参战：F6F地狱猫 + SBD轰炸机 + 航母 + 驱逐舰
- 制空：6 / 5（充足）
- 损失：2张

**结果：**
```
己方火力:
  ...
  制空: 6 / 5 ✓ 空军可返航

战损情况:
  - F6F地狱猫战斗机 被击中但成功返航 ✈️
  - SBD无畏式俯冲轰炸机 被击中但成功返航 ✈️
```

**解释：**
- 制空充足，触发返航能力
- 空军进入弃牌堆（而非商店）
- 可在下回合重新抽到

### 场景4：空军坠毁

**配置：**
- 参战：F4F野猫 + TBD鱼雷机 + 航母
- 制空：2 / 5（不足）
- 损失：2张

**结果：**
```
己方火力:
  ...
  制空: 2 / 5

战损情况:
  - F4F野猫战斗机 被击中并坠毁 ✈️💥
  - TBD鱼雷机"破坏者" 被击中并坠毁 ✈️💥
```

**解释：**
- 制空不足，返航能力无法触发
- 空军返回商店，需重新购买

## 技术要点

### 1. 随机性保留

损失计算仍然是随机的：
- 每次战斗结果不同
- 重甲单位被击沉是概率事件
- 参战单位越多，单个重甲单位被击沉概率越低

### 2. 数据流

```
calculateLosses()
  ↓ 返回 { lostCardIds, damageDetails }
resolveCombat()
  ↓ 传递 damageDetails
getCombatSummary()
  ↓ 生成报告文本
GameBoard.jsx
  ↓ alert() 显示
```

### 3. 兼容性

保留了向后兼容：
```javascript
if (damageDetails && damageDetails.length > 0) {
  // 使用新格式
} else if (lostCards && lostCards.length > 0) {
  // 兼容旧格式
}
```

## 测试建议

### 单元测试重点

1. **重甲单位**
   - 被选中1次 → 未击沉
   - 被选中2次（重甲1） → 击沉
   - 被选中3次（重甲2） → 未击沉

2. **空军返航**
   - 制空充足 + 有返航能力 → 返航
   - 制空不足 + 有返航能力 → 坠毁
   - 没有返航能力 → 正常损失

3. **组合场景**
   - 重甲 + 幸运
   - 空军 + 防空不足
   - 多个重甲单位

### 手动测试步骤

1. 访问 http://localhost:5178/pacific-war-game/
2. 购买并部署战列舰和空军单位
3. 执行多次战斗，观察报告变化
4. 验证描述与实际结果一致
