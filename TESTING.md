# 测试指南

## 最新改进验证清单

### 1. ✅ 补给卡逻辑修复

**测试步骤**：
1. 开始新游戏
2. 自动进入行动阶段后，点击手牌中的"初级补给"卡
3. **预期结果**：
   - 补给数量立即 +1
   - 卡牌直接进入弃牌堆（不进入部署区）
   - 弃牌堆显示这张卡

### 2. ✅ 自动阶段推进

**测试步骤**：
1. 开始新游戏，观察游戏流程
2. **预期结果**：
   - **准备阶段**：自动重置横置卡牌（0.5秒后自动进入抽卡阶段）
   - **抽卡阶段**：自动抽5张牌（0.5秒后自动进入行动阶段）
   - **行动阶段**：需要手动操作，显示"下一阶段"按钮
   - **战斗阶段**：需要手动操作，显示战斗控制面板（不显示"下一阶段"按钮）
   - **购买阶段**：需要手动操作，显示"下一阶段"按钮
   - **弃牌阶段**：自动弃牌和清理（0.5秒后自动进入下一回合）

3. **验证点**：
   - 准备、抽卡、弃牌阶段不显示"下一阶段"按钮
   - 只有行动和购买阶段显示"下一阶段"按钮
   - 战斗阶段显示战斗专用按钮

### 3. ✅ 弃牌堆显示优化

**测试步骤**：
1. 游戏进行几个回合，让弃牌堆积累一些卡牌
2. 观察弃牌堆区域
3. **预期结果**：
   - 只显示最后加入的一张卡牌
   - 卡牌数量正确显示（例如：(8)）
   - 可以悬停查看该卡牌详情

4. 点击弃牌堆区域
5. **预期结果**：
   - 弹出弹窗显示所有弃牌
   - 弹窗标题为"弃牌堆"
   - 所有卡牌可见且可以悬停查看详情
   - 点击"关闭"按钮或点击背景关闭弹窗

### 4. ✅ 抽牌堆显示优化

**测试步骤**：
1. 观察抽牌堆区域
2. **预期结果**：
   - 只显示卡牌数量（例如：(15)）
   - 不显示任何卡牌
   - 无法点击查看内容
   - 保持神秘性，玩家不知道下一张抽什么

## 问题排查

### 补给卡不起作用
- 检查：战斗力是否为0
- 检查：重新部署费用是否为0
- 查看：弃牌堆是否增加了卡牌

### 阶段不自动推进
- 检查：是否在正确的阶段（行动/战斗/购买需要手动）
- 刷新页面重试

### 弹窗无法打开
- 确保点击的是弃牌堆区域
- 检查浏览器控制台是否有错误

### 开发服务器无法访问
```bash
# 重启开发服务器
pkill -f vite
npm run dev
```
