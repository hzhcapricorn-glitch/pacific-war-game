# DBG 卡牌构筑游戏 - 项目计划与实现文档

## 项目目标

通过建立一个 GitHub Pages 托管的网页小游戏，测试 coding agent 的能力。

## 游戏基本功能

1. ✅ 通过 GitHub Pages 托管，可用网页分享
2. ✅ 无需后端存储，关闭网页=结束游戏
3. ✅ 用户界面交互使用鼠标指向和点击

## 技术实现

### 技术栈
- **React 18** + **Vite** - 前端框架和构建工具

- **GitHub Actions** - 自动化部署

## 游戏设计

### 1. 游戏概念

这是一个卡牌构筑（DBG）游戏，类似于”巴巴罗萨”。通过卡牌的构筑、抽卡和使用来完成特殊任务，最终记录回合数和统计信息。

#### 1.1 卡牌类型

- **战斗卡牌** ✅ - 主要卡牌，包含购买价格、重新部署价格、战斗力、特殊能力
  - 实现：10种卡牌（初级补给、步兵班、装甲师、炮兵连、特种部队、后勤军团、侦察单位、补给站、补给仓库、机械化步兵）
- **任务卡牌** ✅ - 记录任务目标、奖励、战斗损失
  - 实现：5张任务卡（前哨战→夺取补给线→防御战→攻坚战→决战）
- **环境卡牌** ⏳ - 未来实现，用于突发事件和战场增益减益
- **角色卡牌** ⏳ - 未来实现，本局战场增益

#### 1.2 卡牌能力系统

实现的能力类型：
- `supply` - 提供补给
- `draw` - 抽取卡牌
- `combat_boost` - 战斗力加成
- `max_supply` - 增加最大补给保留

#### 1.3 资源系统 - 补给

✅ 补给是游戏中的核心资源，用于：
- 购买新卡牌（商店）
- 重新部署横置的卡牌
- 初始最大补给保留限度为 0
- 通过卡牌能力或任务奖励增加保留限度
- 弃牌阶段清除超限补给

#### 1.4 卡牌状态

✅ 实现的卡牌状态：
- **Ready（重置）** - 可用状态
- **Tapped（横置）** - 使用后或战斗后的状态，需要支付重新部署价格来重置

### 3. 游戏流程

#### 3.1 开局初始化 ✅

- 创建5张”初级补给”卡放入弃牌堆
- 生成任务卡堆栈（5张，只显示最上面一张）
- 生成商店（从卡牌池中随机选择，每种10张）
- 归零所有统计数字

#### 3.2 每回合流程

**3.2.0 准备阶段** ✅
- 🔄 **自动执行**（不需要用户输入）
- 重置所有部署区的横置卡牌为ready状态
- 自动进入下一阶段

**3.2.1 抽卡阶段** ✅
- 🔄 **自动执行**（不需要用户输入）
- 从抽牌堆抽5张卡牌到手牌
- 如果抽牌堆为空，将弃牌堆洗牌后放入抽牌堆继续抽
- 如果两个牌堆都为空，停止抽取
- 自动进入行动阶段

**3.2.2 行动阶段** ✅
- **需要用户操作**
- 点击手牌打出卡牌：
  - 🔄 补给卡（如初级补给）：直接获得补给，卡牌进入弃牌堆（即将修复）
  - ✅ 战斗卡：支付费用（如有），卡牌移动到部署区，触发能力
- 点击部署区横置的卡牌：支付重新部署价格，重置为ready状态
- 手动点击”下一阶段”进入战斗阶段

**3.2.3 战斗阶段** ✅
- **需要用户操作**
- 显示战斗控制面板
- 点击部署区的ready卡牌选择参战（可多选）
- 显示己方战斗力 vs 需要战斗力
- 选项：
  - **开始战斗**：
    - 计算总战斗力，对比任务要求
    - 胜利：获得奖励，移除任务卡
    - 失败：任务卡保留
    - 无论胜负：随机损失参战卡牌，其余参战卡牌横置
    - 检查是否完成最后一张任务（游戏结束）
  - **跳过战斗**：直接进入购买阶段
- 战斗结算后自动进入购买阶段

**3.2.4 购买阶段** ✅
- **需要用户操作**
- 点击商店卡牌购买（需要足够补给）
- 购买的卡牌进入弃牌堆
- 手动点击”下一阶段”进入弃牌阶段

**3.2.5 弃牌阶段** ✅
- 🔄 **自动执行**（不需要用户输入）
- 所有手牌自动弃入弃牌堆
- 清除超过最大保留限度的补给
- 回合数+1
- 自动进入下一回合的准备阶段

#### 3.3 游戏结束 ✅

触发条件：
- 完成所有5个任务卡

显示内容：
- 总回合数
- 胜利战斗次数
- 打出卡牌数
- 购买卡牌数
- “重新开始”按钮

## 游戏平衡数据

### 初始设定
- 起始卡牌：5张”初级补给”（每张提供1补给）
- 起始补给保留：0
- 每回合抽牌：5张

### 卡牌价格范围
- 补给卡：0-5
- 战斗单位：2-6
- 功能卡：3-6

### 任务难度曲线
1. 前哨战：需要战斗力5
2. 夺取补给线：需要战斗力8
3. 防御战：需要战斗力10
4. 攻坚战：需要战斗力15
5. 决战：需要战斗力20
