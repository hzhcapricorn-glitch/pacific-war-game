# 更新日志

## [1.2.0] - 2026-04-21

### 新增功能 ✨

#### 卡牌竖向显示
- 卡牌改为竖向布局，更符合传统卡牌游戏
- 移除横置时的旋转90度效果
- 横置状态改为灰度滤镜 + 半透明显示

#### 商店堆叠系统
- 商店卡牌按种类分组堆叠显示
- 每种卡牌显示剩余数量徽章（右上角金色圆形）
- 视觉堆叠效果：使用 CSS 伪元素创建三层卡牌
- 购买时自动从对应堆中取一张

#### 弃牌堆状态优化
- 所有进入弃牌堆的卡牌状态重置为 ready
- 弃牌堆显示的卡牌始终竖立（不横置）
- 包括补给卡、战斗后弃牌、回合结束弃牌

### 游戏规则调整 🎮

#### 战斗损失机制改变
- **变更前**：战斗损失的卡牌进入弃牌堆
- **变更后**：战斗损失的卡牌返回商店
- 影响：损失的卡牌可以重新购买，增加策略深度

### Bug 修复 🐛
- 修复补给卡点击无响应问题（instanceId 匹配）
- 修复卡牌移动逻辑的 ID 查找

### 技术改进 🔧
- 优化 `moveCard` 函数支持 instanceId 和 id 双重查找
- 完善商店系统的分组和堆叠逻辑
- 清理调试日志代码

## [1.1.0] - 2026-04-21

### 新增功能 ✨

#### 自动阶段推进
- 准备阶段自动执行卡牌重置
- 抽卡阶段自动抽取5张卡牌
- 弃牌阶段自动弃牌和清理补给
- 只在需要玩家操作的阶段（行动、战斗、购买）暂停
- 每个自动阶段有0.5秒延迟，让玩家看到阶段变化

#### 弃牌堆查看功能
- 弃牌堆只显示最后加入的一张卡牌
- 点击弃牌堆打开弹窗查看所有弃牌
- 弹窗中的卡牌支持悬停查看详情
- 优雅的弹窗动画效果

#### 抽牌堆优化
- 抽牌堆只显示卡牌数量
- 不显示具体卡牌，保持抽牌的神秘感
- 防止玩家提前知道下一张抽什么

### 改进 🔧

#### 补给卡逻辑修复
- 修复"初级补给"等补给卡无响应的bug
- 补给卡（战斗力为0且重新部署费用为0）直接使用后进入弃牌堆
- 不再占用部署区空间
- 立即获得补给效果

#### UI 优化
- "下一阶段"按钮只在需要时显示（行动、购买阶段）
- 战斗阶段使用专用的战斗控制按钮
- 弃牌堆区域可点击，有悬停提示
- 添加弹窗组件和相关样式

### 文档更新 📚
- 更新 `plan.md` 反映当前实现状态
- 添加 `TESTING.md` 测试指南
- 更新 `README.md` 说明新功能
- 添加 `CHANGELOG.md` 记录变更

## [1.0.0] - 2026-04-21

### 首次发布 🎉

#### 核心功能
- 完整的卡牌构筑游戏机制
- 10种战斗卡牌，各具特色
- 5张任务卡，难度递增
- 战斗系统：选择卡牌、计算战斗力、结算奖励和损失
- 商店系统：使用补给购买新卡牌
- 补给管理：补给上限、超限清除

#### 游戏流程
- 6个游戏阶段：准备→抽卡→行动→战斗→购买→弃牌
- 完整的回合循环
- 自动洗牌和牌堆管理
- 卡牌横置和重置机制

#### UI/UX
- 响应式游戏布局
- 卡牌缩略和详细视图
- 悬停预览功能
- 流畅的CSS动画
- 选中状态高亮
- 游戏结束统计界面

#### 技术实现
- React 18 + Vite
- React Context 状态管理
- 模块化游戏系统（CardEngine, CombatSystem, ShopSystem）
- JSON数据驱动的卡牌定义
- GitHub Actions 自动部署
