# 最新更新说明 (v1.2.0)

## ✨ 新增改进

### 1. ✅ 卡牌改为竖向显示

**变更内容**：
- 卡牌从横向改为竖向布局
- 移除了横置时的旋转90度效果
- 横置卡牌现在通过灰度和半透明显示状态

**视觉效果**：
- 卡牌尺寸：100-120px 宽 x 140px+ 高
- 更符合传统卡牌游戏的展示方式
- 横置卡牌：灰度 + 60% 不透明度 + 灰色边框

### 2. ✅ 商店卡牌堆叠显示

**变更内容**：
- 商店卡牌按种类分组堆叠
- 每种卡牌显示剩余数量
- 购买时从对应堆中取一张

**实现细节**：
- 堆叠效果：使用 CSS `::before` 和 `::after` 伪元素
- 数量徽章：右上角金色圆形徽章显示剩余数量
- 购买逻辑：自动找到对应种类的第一张卡牌实例

**文件位置**：
- 逻辑：[src/components/Shop.jsx](src/components/Shop.jsx)
- 样式：[src/styles/cards.css](src/styles/cards.css) `.shop-card-stack`

### 3. ✅ 弃牌堆卡牌不横置

**变更内容**：
- 所有进入弃牌堆的卡牌状态重置为 `ready`
- 弃牌堆显示的卡牌始终保持竖立

**影响范围**：
- 打出补给卡 → 弃牌堆（ready）
- 弃牌阶段手牌 → 弃牌堆（ready）
- 战斗损失 → 商店（见下一条）

**实现位置**：
- [src/core/GameState.jsx:120-128](src/core/GameState.jsx#L120-L128) - PLAY_CARD
- [src/core/GameState.jsx:293-296](src/core/GameState.jsx#L293-L296) - END_TURN

### 4. ✅ 战斗损失的卡牌返回商店

**规则变更**：
- **之前**：战斗损失的卡牌进入弃牌堆
- **现在**：战斗损失的卡牌返回商店（可重新购买）

**游戏影响**：
- 损失的卡牌不会永久消失
- 可以用补给重新购买损失的卡牌
- 增加了策略深度：损失强力卡牌后仍有机会购回
- 商店卡牌种类会动态变化

**实现位置**：
- [src/core/GameState.jsx:259-275](src/core/GameState.jsx#L259-L275) - RESOLVE_COMBAT

## 🎮 游戏机制更新

### 补给卡使用流程
1. 点击手牌中的补给卡（如"初级补给"）
2. 立即获得补给 +1
3. 卡牌直接进入弃牌堆（状态：ready）
4. 显示补给增加动画

### 战斗卡使用流程
1. 点击手牌中的战斗卡
2. 卡牌移动到部署区
3. 触发卡牌能力（如有）
4. 可参与战斗

### 战斗损失机制
1. 战斗结算时，根据任务卡指示随机损失参战卡牌
2. 损失的卡牌返回商店（状态重置为 ready）
3. 其他参战卡牌横置（tapped）
4. 商店显示更新：对应堆的数量 +1

### 商店购买机制
1. 商店按卡牌种类显示（每种显示为一个堆）
2. 每个堆显示剩余数量徽章
3. 点击卡牌堆购买一张
4. 购买后：补给 -cost，卡牌进入弃牌堆，数量 -1
5. 数量为 0 的堆自动隐藏
